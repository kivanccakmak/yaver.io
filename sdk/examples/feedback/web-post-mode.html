Post Mode BATCH

Record everything offline. Submit compressed bundle when done. Agent analyzes afterwards.

import { YaverFeedback } from '@yaver/feedback-web';

// Post Mode: no streaming, pure offline recording
YaverFeedback.init({
  trigger: 'floating-button',
});

// In this mode:
// 1. User clicks "Start Recording"
// 2. Screen + mic captured locally (no network traffic)
// 3. User tests the app, narrates bugs verbally
// 4. User takes screenshots at key moments
// 5. User clicks "Stop & Send"
// 6. SDK compresses video (H.264/VP9), audio (AAC/Opus)
// 7. Builds metadata: timeline, screenshots, device info, console errors
// 8. Uploads compressed bundle to agent via multipart POST
// 9. Agent processes everything at once
// 10. Agent generates fix prompt from the full session
//
// Best for: slow connections, detailed testing sessions, QA workflows
// Bundle size: ~2-5 MB per minute of recording (720p)
//
// On the agent side:
//   yaver feedback list          — see the report
//   yaver feedback show      — view timeline + transcript
//   yaver feedback fix       — create AI task from the report

What gets captured (no streaming)

  • Screen recording (WebM/VP9 or MP4/H.264)
  • Microphone audio (voice narration)
  • Screenshots with annotations
  • Console errors (automatic)
  • Mouse/tap events timeline
  • Page URL, browser info, viewport size
  • All compressed before upload
